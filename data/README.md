# Data

No raw data is stored in this repository. Both inputs are public.

## 1. LaDe last-mile delivery dataset (CC BY-NC 4.0)

Source: [Cainiao-AI/LaDe on Hugging Face](https://huggingface.co/datasets/Cainiao-AI/LaDe)

We use the **pickup** records (`pickup_{city}.csv`) of five cities
(Shanghai, Hangzhou, Chongqing, Yantai, Jilin; Jilin has too few regions and
is excluded from the LOCO pool). Download:

```bash
pip install huggingface_hub
python - <<'EOF'
from huggingface_hub import snapshot_download
snapshot_download(repo_id="Cainiao-AI/LaDe", repo_type="dataset",
                  allow_patterns="pickup/*.csv", local_dir="LaDe_raw")
EOF
```

Preprocessing (region aggregation, hourly binning, ramp-up trimming) is done
on the fly by `src/data/build_dataset.py`; point `data.lade_dir` in
`configs/default.yaml` at the download directory. The region ordering
follows first-appearance order in the CSVs, which keeps our tensors aligned
row-by-row with the IMPEL embeddings below.

**License note**: LaDe is distributed under CC BY-NC 4.0 by its authors.
This repository does not redistribute it; users must obtain it from the
source above and comply with that license.

## 2. LLM geographic region embeddings (IMPEL pipeline)

Source: [github.com/tongnie/IMPEL](https://github.com/tongnie/IMPEL)

We reuse the publicly released Llama-3 region embeddings
(`llmvec_llama3_Delivery_{CITY}.npy`, 4096-d per region) generated by the
IMPEL authors from region addresses and POI text. Clone their repository and
point `data.llm_emb_path` in `configs/default.yaml` at its `data/` folder.
`scripts/check_align.py` verifies that region counts align with our
pipeline.
