# Zero-Shot Cross-City Delivery Demand Forecasting with Online Conformal Calibration

Code and configurations for a staged forecasting framework (**Z2C**, zero-shot
to calibrated) for last-mile delivery demand in newly opened cities:

- **Strict city-level zero-shot protocol (LOCO)**: leave-one-city-out; the
  target city has *zero* historical observations at day 0.
- **Inductive spatio-temporal quantile backbone**: identity-free node
  representations from static region attributes + offline LLM geographic
  embeddings; kNN graph built at inference time; TCN encoder with quantile
  heads.
- **Staged conformal calibration**: source-calibrated split conformal at
  day 0, full-recomputation online conformal calibration from day 1
  (nominal 90% coverage restored within 1–3 operating days).
- **Online ensemble** of the network and accumulating target climatology for
  point accuracy.
- **Newsvendor replay** to evaluate calibrated quantiles in units of
  operational cost.

Baselines included: pooled hour-of-week climatology, kNN profiles, weighted
conformal (cosine and Sinkhorn-OT/WR-CP weights), ACI, and IMPEL
partial-observation transfers.

## Reproduce the main tables in three steps

```bash
# 1. Environment (Python >= 3.10, single consumer GPU is enough)
pip install -r requirements.txt

# 2. Data (LaDe is public; see data/README.md for licensing details)
#    - download LaDe pickup CSVs (Cainiao-AI/LaDe on Hugging Face)
#    - generate/download Llama-3 region embeddings via the IMPEL pipeline
#    - set the paths in configs/default.yaml

# 3. Run the full LOCO matrix (5 seeds) and evaluations
bash scripts/run_loco_matrix.sh   # data build + training + all evaluations
bash scripts/run_supplement.sh    # WR-CP baseline + IMPEL LOCO transfers
python tools/make_tables.py       # regenerate all LaTeX tables from results/
python tools/make_figures.py      # regenerate all figures
```

Every number reported in the accompanying manuscript is generated by
`tools/make_tables.py` / `tools/make_figures.py` from the JSON files in
`results/` — no hand-copied values. The exact training recipe (learning rate
5e-4, gradient clipping 1.0, early stopping, seeds 0–4) is fixed in
`configs/default.yaml` and `scripts/run_loco_matrix.sh`.

## Repository layout

```
├── configs/          # experiment configuration (paths, recipe, seeds)
├── data/             # dataset download & preprocessing instructions (no raw data)
├── src/
│   ├── models/       # inductive ST quantile backbone
│   ├── calibration/  # weighted / online conformal
│   ├── decision/     # newsvendor replay
│   ├── data/         # LaDe -> hourly region-level demand pipeline
│   └── *.py          # training, replay, and evaluation entry points
├── scripts/          # batch scripts that reproduce every experiment
├── results/          # result JSONs used by the paper (tables/figures inputs)
├── tools/            # table and figure generators
└── LICENSE           # MIT (code); data licenses apply separately
```
